Lesson 4: Propositional Modelling

Modelling a System

System modelling in propositional logic is using logical statements to reason about a system. The system components can be represented as logical statements and then logical equivalence and truth tables can be used to fully analyse the system and optimise its performance. This approach can help in modelling complex systems and determine potential issues before they arise.

Practical Example:

If the capacity of the lift is exceeded then the doors remain open. The doors are closed when the lift is moving. The alarm is on whenever the doors are closed and the capacity is exceeded, or the lift is moving with the doors open.

First label the propositions as below:

  • E     The capacity of the lift is exceeded
  • C     The lift's doors are closed
  • M     The lift is moving
  • A     The alarm is on

Now using logical connectives:

  • If the capacity of the lift is exceeded then the doors remain open
    • E => ¬C
  • The doors are closed when the lift is moving.
    • M => C
  • The alarm is on whenever the doors are closed and the capacity is exceeded, or the lift is moving with the doors open.
    • (C & E) | (M & ¬C) => A

Compound Propositions:

  • E => ¬C                              If the capacity of the lift is exceeded then the doors remain open.
  • M => C                               The doors are closed when the lift is moving.
  • (C & E) | (M & ¬C) => A     The alarm is on whenever the doors are closed and the capacity is exceeded, or the lift is moving with the doors open.

Feasibility

  • The requirements are not feasible if the propositions are contradictory (that is they are never all true at the same time).
  • If we have a contradiction, then every row of the truth table for (E => ¬C) & (M => C) & [(C & E) V (M & ¬C) => A] will result in false.
  • To show that a system is feasible we must demonstrate that at least one row of the table does not result in false.

Terminology and consistency

  • A proposition is said to be a tautology if it always has the truth value “T”.
  • A proposition is said to be a contingency if it sometimes has truth value “T” and sometimes “F”.
  • A proposition is said to be an inconsistency (contradiction)it has the truth value “F”.
  • A consistent proposition is one which is a tautology or a contingency.
  • We can establish which of these classes a proposition falls into just by constructing its truth table.

Exercise 5:

  • If my program is correct,
    it will pass the validation test.
    My program passes the validation test.
    Therefore, my program is correct.
  • Let, C be ‘My program is correct’, and
    V be ‘My program passes the validation test’.
  • We can rewrite the above argument
    • [(C => V) & V] => C
    • This is the proposition that must be a tautology for the argument to be valid.
