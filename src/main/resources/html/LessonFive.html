Lesson 5: Propositional Proof and Deduction

Valid Arguments

  • An argument is a relationship between a set of propositions P1, P2, ...Pn, called premisses and another proposition Q, called the conclusion.
  • If the proposition P1 & P2 & ... & Pn => Q is a tautology, we also write,
    • P1 & P2 & ... & Pn ⊨Q
    • Which is pronounced ‘P1 to Pn semantically entail Q’. (This means that by checking all the possibilities we can show that whenever P1 to Pn are true, Q is also true.)

Deriving System Properties

  • To find all the system properties we could form the conjunction of the requirements and construct the truth table.
  • Then we could give an interpretation of each row.
  • All the rows that result in true are possible outcomes, all those false cannot occur.
  • Unfortunately this process has complexity and it becomes impractical.
  • The way forward is to identify those properties that are of most concern and then deduce their correctness.
  • The lift system property:
    • In normal operation the lift will not move if the capacity is exceeded.
    • The task is to
      1. Formulate the problem
      2. Solve it

1. Formulating the problem

  • We consider problems of the form
    • Given some premises deduce some conclusion
  • The lift problem is
    • Given the lift is in normal operation
    • Deduce the lift will not move if the capacity is exceeded.
What do we mean by the lift is in normal operation?
  • Take this to mean that the requirements are satisfied: in particular we can assume E => ¬C and M => C are true.
  • The conclusion can be written as the proposition E => ¬M .
  • So the problem now looks like this
    • Given E => ¬C, M => C deduce E => ¬M
  • If the capacity of the lift is exceeded then the doors remain open. The doors are closed when the lift is moving. The lift will not move if the capacity is exceeded.

2. Solving the problem: Two approaches

  • There are two approaches to demonstrate this property.
  • Semantic consequence: This approach consider all the possible interpretation of propositions and show that when the premises are true then the conclusion is also true.
  • Proof: This method uses rules of deduction to provide an argument that starts from the premises and leads to the conclusion.

Semantic Consequence

  • A conclusion is a semantic (or logical) consequence of its premises if
    • There is no interpretation where the conclusion is false and all the premises are true.
  • Or looked at the other way round
    • When all the premises are true then the conclusion must also be true.
  • If C is a semantic consequence of the premises P, Q, R we write P, Q, R ⊨ C.
    • P ⊨ C means C is a semantic consequence of P (it is true if all of P are true)
  • The symbol ⊨ is called the semantic turnstile.
  • We call the expression P, Q, R ⊨ C a semantic sequent. We never speak of the truth of a sequent but instead talk about its validity.

Let us use the first approach to deduce the validity of the lift problem:

    [(E => ¬C), (M => C)] ⊨ (E => ¬M)
E C M ¬C E => ¬C M => C ¬M E => ¬M

However, to some this process might seem complex and daunting.

We can try digging into proof by deduction and see if that is simpler.

Proof by Deduction

Proof by deduction (or reasoned argument) is like a game. At the start, we are given a resource pack, and the task is to reach some end point. The game has a set of rules which tell us how we can move. We must reach our goal using legal moves only.

  • In propositional calculus the starting pack is a set of premises: those propositions that we assume are true; the end point is the conclusion; the rules of the game are the propositional equivalences and the rules of deduction.
  • You have been supplied with a page of these laws and rules for reference in (Equivalences and Rules of Deduction). The list is not complete, but sufficient for the needs of this module.
  • The equivalences allow us to replace one propositional expression with another.
  • The laws of deduction allow us to deduce the truth of a proposition from the truth of earlier ones.

Rules of Deduction

  • & - introduction     (& - Intr)
    • P, Q ⊨ P & Q
    • Given P is true and Q is true,
    • deduce P & Q is true
  • & - elimination     (& - Elim 1)
    • P & Q ⊨ P
    • Given P and Q are both true,
    • deduce P is true
  • | - introduction     (| - Intr 1)
    • P ⊨ P | Q
    • Given P is true,
    • deduce P or Q is true
  • | - elimination     (| - Elim 1)
    • P | Q, P => R, Q => R ⊨ R
    • Given that at least one of P or Q is true,
    • and that if P is true then so is R,
    • and that if Q is true then so is R,
    • deduce R is true
  • => - elimination     (=> - Elim)
    • P => Q, P ⊨ Q
    • Given that if P is true then so is Q,
    • and that P is indeed true,
    • deduce Q is true

Reasoning Example

This last law is typical of everyday reasoning.

  • P => Q, P ⊨ Q
  • If I oversleep, then I miss my 9 o'clock lecture.
  • I have overslept.
  • Therefore, I miss my 9 o'clock lecture.

Definition of Proof

  • A proof is a sequence of steps
  • Each step is an application of a rule of deduction.
  • The final step gives the conclusion.

Presentation of Proof

We have to present proofs in a way which helps other people follow them. The standard method gives each line in the following way:

Number                   Proposition                   Rule

A proof is given as a sequence of lines. Each line is numbered for reference. On each line put the proposition that is deduced from those above. On the right-hand side of the line put the line numbers and rule that was used. A given proposition is called a hypothesis or premise.

Example

Given A => B, ¬B prove ¬A

  1. ¬B
  2. A => B
  3. ¬A | B
  4. ¬B & (¬A | B)
  5. (¬B & ¬A) | (¬B & B)
  6. (¬B & ¬A) | False
  7. ¬B & ¬A
  8. ¬A
