Lesson 1: Introduction to Propositional Logic

Logic plays an important role in all sciences and especially so in computing:

  • The flow of control in a program depends on the result of logical expressions in branching conditions (IF, WHILE…)
  • Computer architecture is based on binary arithmetic (1’s and 0’s). These digits are implemented by deciding whether an electrical charge is high or low, or in logical terms true or false.
  • Logic provides a set of rules for reasoning about a system.
  • To prove some property we must first express it symbolically, and then give a step by step argument to move from our starting point to the conclusion.
  • Each step has to be justified: it must be logical.
  • It must be an application of a rule of deduction.
  • There are many sets of rules for reasoning:
    • Different logics for different systems.
    • We shall be dealing with propositions and predicates.
    • This is the logic of sentences that can take one of two values: true or false

To reason about any system there needs to be an agreed set of rules. Logic is the study of deductive arguments.

What is a proposition?

  • In natural English, a proposition is a declarative sentence in the form of a statement, which may be either true or false. We may not know which of these values it has, but we can tell it must have one of them.
  • If a proposition is true, then we say it has a truth value of "true"; if a proposition is false, its truth value is "false".
  • Examples of propositions:
    • Grass is green
    • Two plus two is four
    • It is raining today
    • 2 + 5 = 5
    • King Henry the Eighth had six wives

Exercise 1:

Connectives

Compound propositions can be built from simple propositions by using logical connectives.

Any proposition may be constructed out of simple propositions using the connectives we consider.

Note that the set of connectives we will use is not the only one possible, though it is popular one.

The connectives are:

Name Symbol Description
Negation ¬/! Not
Conjunction & And
Disjunction | Or
Implication => If...Then
Equivalence <=> If and only if
Unary Operators Binary Operators

Example

Let P be the proposition the house is white.

Let Q be the proposition the garage is green.

Then


¬P the house is not white
P & Q the house is white and the garage is green
P | Q the house is white or the garage is green
P => Q if the house is white then the garage is green
P <=> Q the house is white if and only if the garage is green
